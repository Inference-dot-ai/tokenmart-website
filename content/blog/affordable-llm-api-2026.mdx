## 7 Tips for using an affordable LLM API

To make the most of an **affordable LLM API**, apply practical optimizations that reduce token use and increase throughput without sacrificing quality.

### Tip 1: Prompt and response optimization
- Keep prompts concise, use system messages to set defaults, and truncate irrelevant context.
- Limit response length with max_tokens and guardrails.

### Tip 2: Caching and reuse
- Cache completions for identical prompts or near-duplicate queries.
- Use fingerprinting (hashes) to detect reusable responses.

### Tip 3: Hybrid routing and model selection
- Route light inference to lower-cost models and heavy reasoning to high-capacity models.
- TokenMart supports multi-model routing to optimize cost-performance.

### Tip 4: Token budgeting and alerts
- Set daily token budgets and alert thresholds.
- Throttle non-critical features when approaching limits.

### Tip 5: System messages and few-shot examples
- Use system prompts to reduce the need for repeated in-prompt instruction.
- Store few-shot examples externally and fetch only when necessary.

### Tip 6: Post-processing vs generation
- Move deterministic formatting tasks (e.g., truncation, sorting) to post-processing to save tokens.
- Use small local models for formatting or validation where possible.

### Tip 7: Negotiate committed use
- Commit to monthly or annual token plans to unlock price breaks.
- TokenMart’s bulk plans often provide ~20% savings vs on-demand rates.

Apply these tactics together: each reduces token consumption or shifts work to lower-cost layers, multiplying savings over time.
